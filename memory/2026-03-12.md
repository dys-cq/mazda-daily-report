# 2026-03-12 - Mazda KPI Report Fixes

## CSI 报表修复

### 问题 1: 评价工单结算范围显示错误
- **现象**: "评价工单结算范围"显示的是大区值（如"西区"），而不是时间范围
- **原因**: CSI 统计表列头格式为"字段名：时间范围"，行内数据第一列是经销商代码，第二列是经销商名称，后续列包含大区值
- **修复**: 从列头解析出时间范围（如"2 月 1 日 -3 月 8 日"），而不是取行内大区值

### 新增指标
1. **直评日期范围**: 从统计表头"直评日期：xxxx-xxxx"解析
2. **本月维修合同数**: 从统计表头"本月维修合同：xxxx-xxxx"解析，带时间范围后缀

### 问题 2: HTML 报表未渲染
- **现象**: HTML 文件打开显示的是 Markdown 源码，没有表格渲染
- **原因**: 之前用`<pre>{md_text}</pre>`直接包裹 Markdown 原文
- **修复**: 改为结构化 HTML 渲染，使用真正的`<table>`、`<tr>`、`<td>`标签，添加 CSS 样式（卡片布局、蓝色标题、表格边框）

## 文件修改

  - CSI 统计解析逻辑重构，正确映射字段名和数值

## 2026-03-12 13:44 - 数据源限制说明

### 问题
用户反馈报表中：
1. 保险/平台线索统计数据没有
2. 客诉/线索统计数据没有
3. CSI 自主调研统计数据没有
4. 总览图表没有可视化结果

### 根本原因
**售后日报 Excel 文件不包含以下数据**：
- 客诉/线索明细（需要独立的 `客户投诉.csv` 或 HTML 文件）
- CSI 统计表（需要独立的 `CSI*.xlsx` 文件）
- 保险/平台线索月度明细（需要 `保险平台线索 0101-0310.xlsx`，包含 1 月/2 月/3 月 多个 sheet）

**售后日报 Excel 只包含**：
- ✅ 综合日报（经营数据）
- ✅ 零附件明细
- ✅ 客户关怀（回访数据，非投诉）
- ✅ 汇总数据

### 解决方案
需要用户将以下独立文件放入 `E:/2026 年 KPI/每日分析/` 目录：
1. `客户投诉.csv` 或 HTML 线索文件
2. `CSI 统计表.xlsx`
3. `保险平台线索 0101-0310.xlsx`（多 sheet）

### 已创建工具
- `export_latest_data.py` - 从售后日报 Excel 导出 CSV
- `generate_kpi_report.py` - 一键生成报表
- `build_final_report_v7.py` - 修改为从 run_folder 查找 CSV 文件

### 当前可用功能
- ✅ 经营数据统计（服务/零件/工时收入、台次、达成率、机油/事故单车）
- ✅ 零附件指标（当月/季度 目标/达成/达成率）
- ❌ 客诉/线索（缺少源文件）
- ❌ CSI（缺少源文件）
- ❌ 保险/平台（缺少源文件）
- ✅ 图表框架（有 canvas，但无数据）
