# 夜间自动执行 TODO 清单（00:00–08:00）

> 目标：每天稳定产出“可落地、可复用、可迭代”的 Skill 与内容资产。

## 0. 每日目录初始化（00:00）
- [ ] 创建当日目录：`daily-results/YYYY-MM-DD/`
- [ ] 创建子目录：
  - [ ] `original/`
  - [ ] `rewrite/`
  - [ ] `skills/`
  - [ ] `reports/`
  - [ ] `logs/`
- [ ] 写入运行日志头：`logs/run-start.log`

## 1. 信息搜集与候选池构建（00:05–01:30）
- [ ] 采集自动化办公、自媒体、数据分析、自动化提效四类来源
- [ ] 记录候选文章清单（标题、URL、来源、时间）
- [ ] 初筛：去掉广告页、纯观点无步骤页、过时内容
- [ ] 产出候选列表：`reports/candidate-sources.md`

## 2. 高价值筛选（01:30–02:30）
- [ ] 依据标准打分（可落地性、步骤清晰度、复用价值、工具通用性）
- [ ] 每晚确定重点文章（建议 3–8 篇）
- [ ] 记录入选原因与预期可复用点
- [ ] 输出：`reports/selected-articles.md`

## 3. 双文件产出（02:30–05:30）
对每篇入选文章执行：
- [ ] 保存原文：`original/<slug>-original.md`
- [ ] 生成公众号改写：`rewrite/<slug>-wechat.md`
- [ ] 改写包含：
  - [ ] 强标题（结果导向）
  - [ ] 开头钩子（痛点/反差/数据）
  - [ ] 案例化拆解（步骤+工具+时间成本）
  - [ ] 行动清单（可直接照做）
  - [ ] 风险与边界提醒
- [ ] 在日志登记每篇状态（成功/失败/原因）

## 4. Skill 反向沉淀（05:30–06:40）
- [ ] 从当晚文章抽取可技能化流程
- [ ] 每晚至少产出 1–3 个新 Skill 草案
- [ ] 每个草案包含：
  - [ ] 解决问题
  - [ ] 触发条件
  - [ ] 输入/输出
  - [ ] 执行步骤
  - [ ] 边界与注意事项
- [ ] 保存：`skills/<skill-name>.md`

## 5. 每日总报告（06:40–07:20）
- [ ] 汇总当日新增：文章数、改写数、Skill 数
- [ ] 列出最值得复用的 3 条方法
- [ ] 标注失败项与次日修复建议
- [ ] 输出：`reports/daily-summary.md`

## 6. 收尾与归档（07:20–08:00）
- [ ] 写入结束日志：`logs/run-end.log`
- [ ] 校验目录完整性（5 大子目录是否齐全）
- [ ] 校验文件配对完整性（每篇是否 original+wechat 成对）
- [ ] 更新“次夜优先队列”：`reports/next-queue.md`

---

## 执行优先级规则
1. **先保证完整性**：目录、日志、双文件配对必须完整。
2. **再追求数量**：宁可少做，也不做低质量内容。
3. **最后做创新**：在稳定产出的基础上增加新 Skill 创新。

## 质量红线（必须满足）
- 不保存空文件或无效链接。
- 改写稿不得只做同义替换，必须结构重构与场景化增强。
- Skill 草案必须能对应真实场景，且步骤可执行。

## 每日最小交付（保底）
- 原文 ≥ 3 篇
- 改写 ≥ 3 篇
- 新 Skill 草案 ≥ 1 个
- 每日报告 1 份
